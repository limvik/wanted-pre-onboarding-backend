# 요구사항 분석

## 서비스 개요

-   본 서비스는 기업의 채용을 위한 웹 서비스 입니다.
-   회사는 채용공고를 생성하고, 이에 사용자는 지원합니다.

## 요구사항

### 기능

- 회사는 채용공고 등록/수정/삭제
- 회사와 사용자는 채용공고 목록 조회
- 회사와 사용자는 채용공고 키워드 검색(선택)
- 회사와 사용자는 채용공고 상세내용 조회
	- 채용공고에 채용내용 및 회사가 올린 다른 채용공고 id 목록(선택) 포함
- 사용자는 채용공고에 지원(선택)
	- 사용자는 1회만 지원 가능

### 비기능

- Java & Spring
- ORM 사용
- RDBMS 사용

### 기타

- 모델: 회사, 사용자, 채용공고, 지원내역(선택)
- 계정 생성 및 인증 기능 생략(DB 직접 입력)
- 프론트엔드 생략
- 기한 내에 제작 가능하도록 현재 제작 가능한 REST API로 제작
- 국가별 사업자번호 분석 시간 단축을 위해 현재는 국내에 등록된 기업의 채용공고로 한정(해외근무지 채용공고 등록 가능)

## 도메인 모델

- 회사(Company)가 채용공고(Post)를 등록/수정/삭제합니다.
	- 개인적으로는 회사는 별도의 인사담당자를 사용자로 등록 시키는게 더 맞다고 생각되지만, 참고사항에 `회사, 사용자 등록 절차는 생략합니다. (DB에 임의로 생성하여 진행)` 라고 명시된 것을 보아 회사가 하나의 주체로써 동작하기를 원하는 것으로 판단하였습니다.
- 일반사용자(User)가 올라온 채용공고(Post)를 확인하고 지원하면 지원내역(Application)이 기록됩니다.

제시된 요청 및 응답 예제에서 변경하거나 추가한 부분에 대한 설명을 아래 작성합니다.

### 회사(Company)

- 회사 이름(name)은 중복될 수 있으므로, 사람이 회사 식별이 가능하도록 `사업자등록번호(business_number)`를 추가합니다.

> 기업회원이 가입 시 추가 정보를 입력하도록 유도하거나, 사업자등록번호를 이용하여 대표자이름 및 사업장 주소 등의 상세 정보를 확보할 수 있으므로, 현재 요구사항에 맞추면서 최대한 단순화하기 위해 사업자등록번호를 추가하는 결정을 하였습니다.  
	- 근로복지공단_고용 산재보험 가입 현황([링크](https://www.data.go.kr/data/15002150/fileData.do))
	- 국민연금공단_국민연금 가입 사업장 내역([링크](https://www.data.go.kr/data/3046071/openapi.do))
	- 비즈노 OpenAPI([링크](https://bizno.net/openapi))

### 사용자(User)

- 회원가입 및 인증 절차가 요구사항에는 없지만, 있다고 가정했을 때 최소한이라 생각되는 정보를 추가하였습니다.
- 사용자에게 연락할 수 있는 `이메일(email)`, 본인 임을 인증할 수 있는 `비밀번호(password)`를 추가합니다.

### 지원내역(Application)

- 지원내역이라는 객체의 이름에 맞게 지원 현황을 살펴볼 수 있는 정보를 추가하였습니다. 현재 요구사항에서는 필요 없지만, Spring Data JPA/Hibernate 를 배우는 단계라 실험해볼 겸 추가하였습니다.
- 언제 지원했는지 확인하기 위한 `지원일시(applied_at)`, 언제 다음 전형으로 넘어갔는지 확인하기 위한 `갱신일시(updated_at)`, 마지막으로 현재 어느 전형단계에 있는지 확인하기 위한 `상태(status)`를 추가합니다.

### 채용공고(Post)

- 예제에서 `국가`와 `지역`이 나뉘어 있는데, 지원자 입장에서는 실 근무지 위치를 원하기 때문에 `주소(address)`로 통일하였습니다.
